# STM32F103 摩斯密码训练游戏系统

基于 STM32F103ZET6 微控制器的摩斯密码听力训练游戏，采用 STM32Cube HAL 库开发。

## 系统概述

本项目实现了一个交互式摩斯密码训练游戏，玩家通过按键启动题目播放，聆听蜂鸣器发出的摩斯码后，通过串口输入对应的字母答案，系统根据正确与否给予音频反馈并更新得分。

## 硬件规格

| 项目 | 参数 |
|------|------|
| MCU | STM32F103ZET6 (LQFP144) |
| 主频 | 72 MHz (HSE + PLL x9) |
| Flash | 512 KB |
| SRAM | 64 KB |
| 供电 | 3.3V |

## 引脚分配

| 外设 | 引脚 | 功能说明 |
|------|------|----------|
| LED1 | PE5 | 状态指示灯（等待输入时常亮，错误闪烁） |
| Key1 | PE4 | 启动题目播放 / 重复播放（外部中断，下降沿触发） |
| Key2 | PE3 | 系统复位（外部中断，下降沿触发） |
| 有源蜂鸣器 | PB8 | 摩斯码播放发声（高电平响） |
| 无源蜂鸣器 | PB7 (TIM4_CH2) | 正确/错误乐谱播放（PWM） |
| LED_PWM | PB5 (TIM3_CH2) | 渐变呼吸灯效果（TIM6 中断驱动） |
| USART1_TX | PA9 | 串口发送（printf 重定向） |
| USART1_RX | PA10 | 串口接收（中断模式） |
| OSC_IN/OUT | PC14-PC15 | LSE 晶振（32.768 KHz RTC 时钟源） |

## 外设配置

### 时钟树
- HSE 外部晶振 → PLL ×9 → SYSCLK 72 MHz
- APB1 总线 → 36 MHz（定时器 2× 为 72 MHz）
- APB2 总线 → 72 MHz
- RTC 时钟 → LSI（40 KHz，标定后约 40 kHz）

### TIM6（基础定时器）
- 预分频：3600-1
- 重装载值：10000-1
- 中断优先级：4
- 用途：驱动 LED1 呼吸灯渐变效果（PWM 占空比递增/递减）

### TIM3（ PWM 输出）
- 通道：CH2（PB5）
- 预分频：144-1
- 重装载值：10000-1
- 极性：LOW（PWM 脉冲期间为低，LED 亮）
- 用途：LED1 呼吸灯 PWM 占空比输出

### TIM4（PWM 输出）
- 通道：CH2（PB7）
- 预分频：3600-1
- 重装载值：20000-1
- 用途：无源蜂鸣器音调输出，通过设置不同的自动重装值改变频率

### USART1
- 波特率：115200
- 数据位：8
- 停止位：1
- 校验位：无
- 接收模式：中断（1 字节接收，循环缓冲区）

## 摩斯密码游戏机制

### 字母编码表（26 个英文字母 A-Z）

| 字母 | 摩斯码 | 字母 | 摩斯码 | 字母 | 摩斯码 |
|------|--------|------|--------|------|--------|
| A | .- | J | .--- | S | ... |
| B | -... | K | -.- | T | - |
| C | -.-. | L | .-.. | U | ..- |
| D | -.. | M | -- | V | ...- |
| E | . | N | -. | W | .-- |
| F | ..-. | O | --- | X | -..- |
| G | --. | P | .--. | Y | -.-- |
| H | .... | Q | --.- | Z | --.. |
| I | .. | R | .-. | | |

**编码规则：点(dot) = 200ms 蜂鸣声响，划(dash) = 600ms 蜂鸣声响，元素间隔 200ms，字母间隔 300ms。**

### 游戏规则

1. **启动**：按 Key1 从待机状态开始，系统随机选择一个字母（A-Z）作为当前题目
2. **播放**：摩斯码通过有源蜂鸣器播放，LED1 与蜂鸣器同步闪烁
3. **重复播放**：每题最多播放 3 次（Key1 重按）
4. **答题**：通过串口发送一个字母（大小写均可），系统判定正确性
5. **计分**：答对得 10 分，答错不扣分但消耗一次答题机会
6. **重置**：按 Key2 立即停止所有操作，重新初始化游戏，得分清零

### 音频反馈

| 场景 | 音频 | 音符/频率 | 时长 |
|------|------|-----------|------|
| 正确答案 | 成功乐谱 | 523Hz + 659Hz + 784Hz + 1046Hz（1-3-5-i） | 80+80+80+150ms |
| 错误答案 | 错误乐谱 | 370Hz + 349Hz + 196Hz（降四度下行） | 150+300+400ms |
| 答案公布 | LED2 闪烁 | 无 | 5 次 × 300ms |

## UART FIFO 接收机制

采用环形缓冲区（Ring Buffer）实现串口数据的异步接收：

- 缓冲区大小：256 字节
- 写入端：UART 中断回调（`HAL_UART_RxCpltCallback`）
- 读取端：主循环（`uart_fifo_read_all`）
- 防止数据覆盖：仅在 `uart_rx_read != uart_rx_write` 时才消费数据

```
中断 ──写入──▶ [uart_rx_fifo] ──读出──▶ 主循环 ──处理──▶ morse_handle_input()
                  ▲ write                read
                  │                      │
                  └────── 环形索引 ───────┘
```

## 蜂鸣器控制

### 有源蜂鸣器（PB8）
直接 GPIO 输出控制，通过 `HAL_GPIO_WritePin` 拉高/拉低，无源蜂鸣器发声频率固定。

### 无源蜂鸣器（PB7 - TIM4_CH2 PWM）
通过 PWM 占空比（50%）输出可变频率音调：

```
频率 = 72MHz / (预分频3600) / (重装载值+1)
     = 72MHz / 3600 / x
     = 20000 / x  (Hz)

例：BeepPlay(523, 2) → x = 20000/523 ≈ 38
```

### BeepPlay 函数接口

```c
void BeepPlay(int tone, int volumeLevel);
参数：
  tone         - 音调频率（Hz），传 0 表示停止发声
  volumeLevel  - 音量等级（当前固定为 50% 占空比）
```

## 定时器中断时序分析

| 定时器 | 时钟源 | 预分频 | 重装载 | 中断频率 | 周期 |
|--------|--------|--------|--------|----------|------|
| TIM6 | APB1×2 = 72MHz | 3600 | 10000 | 2 Hz | 500ms |
| TIM3 PWM | APB1×2 = 72MHz | 144 | 10000 | 50 Hz | 20ms |
| TIM4 PWM | APB1×2 = 72MHz | 3600 | 动态 | 1~20000 Hz | 50us~1ms |

TIM6 每 500ms 触发一次中断，在中断回调中以步进值 264 递增/递减 PWM 占空比，实现 LED1 的呼吸灯渐变效果。

## 注意事项

1. **防抖处理**：Key1/Key2 采用 500ms 软件防抖（`KEY_DEBOUNCE_MS`），防止按键机械抖动误触发
2. **FIFO 保护**：UART 接收和游戏状态处理分离，中断中仅写入缓冲区，主循环消费
3. **蜂鸣器频率限制**：无源蜂鸣器响应频率范围约 500Hz - 2500Hz，超出此范围音量/音调可能失真
4. **RTC 备用域**：RTC 使用 LSI 时钟（标称 40 kHz），精度低于 LSE（32.768 kHz），断电后时间需重新校准
5. **BOOT0 跳线**：烧录时请将 BOOT0 接高电平，烧录完成后切回低电平从 Flash 启动
